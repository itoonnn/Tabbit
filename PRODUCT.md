# Product

## Register

product

## Users

Tabbit is for small Thai LINE-first groups that want to complete a habit challenge together, starting with friend groups, study groups, small teams, and small communities.

The first wedge is a 5-10 person, 21-day reading challenge where members arrive from a LINE invite, join quickly on mobile, check in daily with minimal effort, see group progress, encourage each other, and return tomorrow.

Primary users:

- Members who want an easy way to keep showing up with the group.
- Organizers who want to start a challenge, invite people, see progress clearly, and avoid manually chasing everyone in chat.

## Product Purpose

Tabbit helps small groups turn shared commitment into group momentum through easy daily check-ins, visible progress, and supportive encouragement.

The MVP exists to validate the core loop:

```text
Create group
-> Set shared habit
-> Daily check-in
-> See group progress
-> Encourage each other
-> Return tomorrow
```

Success means real groups join, complete first check-ins, return on Day 3 and Day 7, encourage each other, and show organizer willingness to run another challenge.

## Brand Personality

Tabbit should feel warm, clear, supportive, focused, and lightly energetic.

It should be friendly enough to invite friends, serious enough for organizers to trust, simple enough to understand in seconds, and action-oriented enough to attract people who actually want to finish a challenge.

The default brand route is Shared Commitment: accountability should feel real without becoming pressure, shame, ranking, or guilt.

## Anti-references

Tabbit should not look or feel like:

- A generic productivity app or to-do list.
- A solo habit tracker with social features added later.
- A leaderboard-first app focused on comparison.
- A shame-based discipline or bootcamp product.
- A passive cozy tracker that is admired but does not create action.
- A cute rabbit app where decoration is clearer than progress.
- A broad community platform or full productivity suite.
- A corporate dashboard with heavy admin complexity.

Avoid UX patterns that publicly shame missed days, make progress feel like exposure, require long onboarding, rely on complex gamification before retention is proven, or make organizers police the group manually.

## Design Principles

1. Group momentum before individual perfection.
   The interface should first show that the group is moving together, not that someone failed.

2. Daily check-in must feel like a few-second action.
   The repeated action should have one clear primary path, optional detail only when it strengthens accountability, and fast feedback after completion.

3. Progress should be visible without explanation.
   Users should quickly understand today's status, group movement, and what their small action contributes.

4. Supportive accountability, never shame.
   Missed-day states should keep people able to return without public failure language, harsh red states, or broken-streak framing.

5. Validate the smallest behavior before expanding scope.
   Delay broad habit types, complex settings, full gamification, advanced dashboards, and identity polish until the first shared challenge loop proves retention.

## Accessibility & Inclusion

Use WCAG AA as the baseline for contrast, readable text, focus states, and touch targets.

Support reduced-motion preferences for animations and state transitions.

Progress and status states must not rely on color alone. Use clear labels, icons, layout, or shape so checked-in, not-yet, missed, and complete states remain understandable for color-blind users and low-attention mobile contexts.

Thai-first copy should be short, natural, and easy to understand in LINE-first mobile usage.
