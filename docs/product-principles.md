# Product Principles

## 1. Group Accountability Over Solo Motivation

**Explanation:** Tabbit ต้องช่วยให้ commitment มีคนรับรู้ ไม่ใช่พึ่งแรงใจคนเดียว

**Product Implication:** ทุก feature สำคัญควรทำให้กลุ่มเห็น commitment, progress หรือ encouragement ได้ชัดขึ้น

## 2. Small Actions Beat Ambitious Plans

**Explanation:** habit ที่เล็กและทำซ้ำง่ายมีโอกาสรอดมากกว่าแผนใหญ่ที่ใช้แรงเยอะ

**Product Implication:** MVP ควรเริ่มจาก shared habit เดียวและ check-in รายวันแบบง่ายที่สุด

## 3. Make Progress Visible

**Explanation:** ผู้ใช้ควรเห็นว่าตัวเองและกลุ่มกำลังขยับไปข้างหน้า

**Product Implication:** Group progress view ต้องอ่านง่าย เห็นสถานะวันนี้และความต่อเนื่องโดยไม่ต้องตีความเยอะ

## 4. Celebrate Consistency, Not Perfection

**Explanation:** เป้าหมายคือกลับมาทำต่อ ไม่ใช่ลงโทษเมื่อพลาด

**Product Implication:** Copy, rewards และ streak design ต้องลดความรู้สึกล้มเหลว และส่งเสริมการเริ่มใหม่

## 5. Reduce Friction For Daily Check-Ins

**Explanation:** check-in คือ core action ถ้ายากเกินไป retention จะตก

**Product Implication:** Daily check-in ควรใช้ไม่กี่วินาที และไม่บังคับกรอกข้อมูลเกินจำเป็น

## 6. Supportive Social Features, Not Shame

**Explanation:** Social pressure ต้องช่วยให้ผู้ใช้รู้สึกมีแรง ไม่ใช่ถูกตัดสิน

**Product Implication:** Encouragement, reactions และ progress display ต้องเน้น support มากกว่า ranking หรือ blame

## 7. Strengthen The Return-Tomorrow Loop

**Explanation:** ทุก feature ต้องช่วยให้ผู้ใช้อยากกลับมา check in อีกในวันถัดไป

**Product Implication:** ถ้า feature ไม่ช่วย retention, accountability หรือ group momentum ให้เลื่อนออกจาก MVP

## 8. Build For One Clear Use Case Before Expanding

**Explanation:** Product ต้องพิสูจน์จาก use case แคบก่อนขยายไปหลายกลุ่มหรือหลาย habit type

**Product Implication:** เริ่มจาก 21-day group reading challenge ก่อนเพิ่ม personal habits, templates หรือ community use cases

## 9. Validate Behavior Before Adding Gamification

**Explanation:** Points, rewards และ leaderboard อาจช่วย motivation แต่ก็อาจสร้างแรงกดดันผิดทาง

**Product Implication:** Gamification ต้องเป็น experiment หลังพิสูจน์ว่า check-in และ encouragement มีพฤติกรรมจริง

## 10. Simplicity Is A Feature

**Explanation:** ความเรียบง่ายช่วยให้กลุ่มเริ่มใช้ได้เร็วและลดภาระของ group leader

**Product Implication:** MVP ต้องหลีกเลี่ยง setup ซับซ้อน, settings เยอะ และ feature ที่ต้องอธิบายยาว
